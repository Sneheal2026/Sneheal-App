═══════════════════════════════════════════════════════════════
  🚀 HOW TO TEST (3 STEPS)
═══════════════════════════════════════════════════════════════

STEP 1: Start Backend
─────────────────────
cd sneheal-backend
npm start

✓ Look for: "Sneheal backend running on port 5000"


STEP 2: Configure Frontend (Real Phone Only)
────────────────────────────────────────────
If testing on emulator: SKIP THIS STEP (already configured)

If testing on real Android/iOS device:
1. Find your PC IP:
   Windows: ipconfig → IPv4 Address (e.g., 192.168.1.8)
   
2. Create frontend/.env:
   EXPO_PUBLIC_API_URL=http://YOUR_PC_IP:5000
   
3. Restart Expo:
   cd frontend
   npm start (stop first if running)


STEP 3: Test the Feature
─────────────────────────
1. Open Sneheal app
2. Log in (JWT required)
3. Go to "Upload Prescription" (from home or settings)
4. Tap Camera or Gallery
5. Select a prescription photo
6. Tap "Tap to Scan"
7. Wait 5-10 seconds
8. See medicine names appear in box!

  🧪 TEST CASES
═══════════════════════════════════════════════════════════════

Good Cases:
  • Clear printed prescription
  • Typed prescription document
  • Handwritten with good lighting

Edge Cases to Test:
  • Very messy handwriting (may return partial results)
  • No prescription in photo (should error gracefully)
  • Blurry photo (error message)
  • Network disconnected (network error shown)
  • Not logged in (auth error)

═══════════════════════════════════════════════════════════════
  ❌ TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

"Network request failed"
→ Backend not running OR wrong EXPO_PUBLIC_API_URL

"Authentication required"
→ Log in to app first

"Gemini API key not configured"
→ Check sneheal-backend/.env has key, restart backend

Empty medicine list
→ Try clearer photo, check backend console logs

"File too large"
→ Image over 10MB, compress or retake

Authorization: Bearer <token>
Content-Type: multipart/form-data
Body: { file: <prescription_image> }

Response:
